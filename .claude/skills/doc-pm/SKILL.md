# Product Manager Documentation Skill

This skill generates high-level product documentation for product managers and stakeholders.

## Instructions

When generating PM documentation, follow the guidelines in SKILL_DOC_PM.md.

### Documentation Process
1. Analyze features and functionality
2. Understand user-facing changes
3. Document product capabilities and use cases
4. Explain business value and impact
5. Generate DOCUMENTATION_PM.md

### Key Principles
- Focus on WHAT the product does and WHY
- Use business language, not technical jargon
- Highlight user benefits and features
- Include metrics and success criteria
- Keep it accessible to non-technical readers

## Supporting Files
The SKILL_DOC_PM.md file contains the documentation standards and templates.
